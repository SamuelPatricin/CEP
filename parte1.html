Resumo sobre Técnicas JavaScript

O que é AJAX?

AJAX (Asynchronous JavaScript and XML) não é uma linguagem de programação, mas uma técnica para solicitar dados de um servidor sem recarregar a página. Ele usa uma combinação de:

  • Um objeto interno do navegador: XMLHttpRequest (para solicitar dados de um servidor web).

AJAX permite que páginas da web sejam atualizadas de forma assincrona, trocando dados com um servidor web nos bastidores. Isso significa que é possível atualizar partes de uma página da web, sem recarregar a página inteira.

Exemplo de AJAX com XMLHttpRequest

    <!DOCTYPE html>
    <html>
    <body>

    <h2>The XMLHttpRequest Object</h2>

    <div id="demo">
        <p>Let AJAX change this text.</p>
        <button type="button" onclick="loadDoc()">Change Content</button>
    </div>

    <script>
    function loadDoc() {
        const xhttp = new XMLHttpRequest();
        xhttp.onload = function() {
            document.getElementById("demo").innerHTML = this.responseText;
        }
        xhttp.open("GET", "ajax_info.txt");
        xhttp.send();
    }
    </script>

    </body>
    </html>

O que é uma Promise?

Uma Promise é uma promessa que uma função faz de te entregar um resultado no futuro. Pode ser um sucesso (resolve) ou um erro (reject). Promises são úteis para operações assincronas, como requisições de dados.

  • Representa um valor futuro: Pode ser um dado, erro, ou qualquer outro valor.
  • Gerencia operações assincronas: Lida com tarefas demoradas sem travar o código.
  • Oferece um jeito mais limpo de lidar com callbacks.

Exemplo de Promise

    <!DOCTYPE html>
    <html>
    <body>
    <h1>JavaScript Promise Object</h1>
    <h2>The then() Method</h2>

    <p id="demo"></p>

    <script>
    function myDisplayer(some) {
        document.getElementById("demo").innerHTML = some;
    }

    let myPromise = new Promise(function(myResolve, myReject) {
        let x = 2;

        if (x == 0) {
            myResolve("OK");
        } else {
            myReject("Error");
        }
    });

    myPromise.then(
        function(value) {myDisplayer(value);},
        function(error) {myDisplayer(error);}
    );
    </script>

    </body>
    </html>

Fetch API

O fetch() método inicia o processo de busca de um recurso de um servidor e retorna uma Promise que é resolvida para um objeto Response.

Exemplo de Fetch API

    <!DOCTYPE html>
    <html>
    <body>
    <h1>JavaScript Fetch API</h1>
    <h2>The fetch() Method</h2>
    <p id="demo">Fetch a file to change this text.</p>

    <script>
    getText("fetch_info.txt");

    async function getText(file) {
        let x = await fetch(file);
        let y = await x.text();
        document.getElementById("demo").innerHTML = y;
    }
    </script>

    </body>
    </html>

Async/Await

Async/Await é uma forma mais simples de escrever código JavaScript que lida com tarefas assincronas, como buscar dados de uma API. A palavra-chave async indica que uma função retorna uma promessa, enquanto await faz com que a execução pause até que a promessa seja resolvida.

Exemplo de Async/Await

    <!DOCTYPE html>
    <html>
    <body>
    <h1>JavaScript async / await</h1>
    <h2 id="demo"></h2>

    <p>Wait 3 seconds (3000 milliseconds) for this page to change.</p>

    <script>
    async function myDisplay() {
        let myPromise = new Promise(function(resolve) {
            setTimeout(function() {resolve("I love You !!");}, 3000);
        });
        document.getElementById("demo").innerHTML = await myPromise;
    }

    myDisplay();
    </script>

    </body>
    </html>

Vantagens e Desvantagens de Técnicas JavaScript

AJAX

  • Vantagens: Experiência de usuário aprimorada, desempenho e interatividade.
  • Desvantagens: Complexidade, problemas de SEO, dependência de JavaScript.

Promises

  • Vantagens: Gerenciamento de operações assincronas, encadeamento de operações e tratamento de erros.
  • Desvantagens: Curva de aprendizado.

Fetch API

  • Vantagens: API padrão moderna, suporte nativo a Promises e flexibilidade.
  • Desvantagens: Suporte limitado em navegadores antigos.

Async/Await

  • Vantagens: Sintaxe limpa, melhor legibilidade, gerenciamento de erros com try/catch.
  • Desvantagens: Suporte limitado em navegadores antigos.

O que é Hoisting?

Hoisting é um comportamento do JavaScript onde as declarações de variáveis e funções são "elevadas" ao topo do escopo (global ou de função), antes de o código ser executado. Isso significa que você pode usar funções e variáveis antes de declará-las no código.

    minhaFuncao(); // Funciona devido ao hoisting
    function minhaFuncao() {
      console.log("Isso é hoisting!");
    }

O que é Desestruturação (Destructuring)?

Desestruturação é um recurso do JavaScript que permite extrair valores de arrays ou propriedades de objetos de forma mais concisa e direta.

Exemplo com Arrays

    const numeros = [1, 2, 3];
    const [a, b, c] = numeros;
    console.log(a, b, c); // 1 2 3

Exemplo com Objetos

    const pessoa = { nome: "João", idade: 30 };
    const { nome, idade } = pessoa;
    console.log(nome, idade); // João 30

O que é Closure?

Closure é um conceito do JavaScript onde uma função "lembra" o ambiente (escopo) em que foi criada, mesmo depois que esse escopo foi encerrado.

    function externa() {
      let contador = 0;
      
      function interna() {
        contador++;
        console.log(contador);
      }
      
      return interna;
    }

    const func = externa();
    func(); // 1
    func(); // 2

Arrow Functions

Arrow functions são uma forma mais curta e legível de definir funções em JavaScript. Elas têm uma sintaxe mais simples e comportamentos especiais em relação ao this e ao objeto arguments.

Exemplo de Arrow Function

    const soma = (a, b) => a + b;

Diferenças entre Arrow Functions e Funções Normais

  • Sintaxe: Arrow functions usam =>, enquanto funções normais usam function.
  • this: Nas arrow functions, this é léxico e herda o valor do contexto onde foi definida.
  • Objeto arguments: Arrow functions não têm acesso ao objeto arguments.
  • Construtores: Arrow functions não podem ser usadas como construtores com new.
  • Retorno implícito: Arrow functions permitem retorno implícito em expressões simples.
